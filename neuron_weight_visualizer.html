🔬 How The Math Becomes a Picture

STEP 1

Start with 784 numbers

Each neuron stores 784 weights in memory. It's just a flat list: [0.02, -0.15, 0.78, ...]. But these aren't random — weight[i] always multiplies pixel[i] from the input image.

STEP 2

Reshape to 28×28

Since MNIST images are 28×28, we can fold the flat list back into a grid. Weight[0] goes to row 0, col 0. Weight[27] goes to row 0, col 27. Weight[28] wraps to row 1, col 0.

STEP 3

Color by value

Positive weight → BRIGHT (neuron EXCITES when pixel is dark here). Negative weight → DARK (neuron INHIBITS when pixel is dark here). Zero → gray (neuron ignores this pixel).

STEP 4

Read the pattern

If the 28×28 image looks like a fuzzy "3" — that neuron fires for digit 3. If it looks like a horizontal bar — it detects horizontal strokes. Each neuron becomes a "feature detector."

🔗 What About Layer 2 and Layer 3?

// Layer 2: 64 neurons, each with 128 weights // These weights can NOT be reshaped into a nice image // because 128 has no meaningful 2D shape — it's not an image dimension Layer 1 weights → reshape to 28×28 → YOU CAN SEE THE PATTERN (784 = 28×28 = the original image shape) Layer 2 weights → 128 numbers → NO meaningful image to show (128 ≠ a square. Each weight connects to a Layer-1 neuron's OUTPUT, not to a pixel. These are abstract features, not visual ones.) Layer 3 weights → 64 numbers → No image, but you CAN see which Layer-2 features each digit-classifier cares about 🎯 KEY INSIGHT: Only Layer 1 weights are directly visualizable because they connect to INPUT pixels. Deeper layers connect to abstract features — you can't draw a picture of "edginess" or "curvature."
