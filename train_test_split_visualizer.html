METHOD 1 — HIGH RISK FOR TIME/USER DATA

🎲 Random Split

Shuffle ALL rows randomly, then split 80/20. Every row has equal chance of landing in test.
METHOD 2 — CORRECT FOR TIME DATA

📅 Time-Based Split

Train on OLDER data, test on NEWER data. The model is tested on the future — like real life.
METHOD 3 — CORRECT FOR USER DATA

👤 Entity-Based Split

All rows for the SAME user stay together — either ALL in training or ALL in test. No user appears in both.

✅ When Random Split is SAFE

  • House Prices: Each house is independent. House #123 doesn't help predict House #456.
  • Images: Each photo is independent (unless same object from different angles).
  • Manufacturing: Each product is independent on the assembly line.

Rule: If rows don't influence each other → random split is fine.

❌ When Random Split is DANGEROUS

  • Stock Prices: Tomorrow's price in training, yesterday's in test = model sees the future.
  • User Data: Same user's Jan data in training, June data in test = model memorizes the user, not the pattern.
  • Medical: Same patient's multiple visits split across train/test = cheating.

Rule: If rows ARE connected (by time or entity) → random split = leakage.
